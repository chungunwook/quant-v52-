# 매일 투자 추천 V5.2 — 실행 가이드

V5.2 퀀트 통합 로직(5단계 공정)을 적용한 주식 자동 추천 시스템입니다.
프록시 서버가 네이버 금융 시세를 파싱하거나 키움 REST API를 중계해서, 가격·종목 입력 없이 자동으로 2종목을 추천합니다.

---

## 파일 구성

| 파일 | 역할 |
|------|------|
| `proxy_server.py` | Flask 프록시 서버 — 키움 API 중계 + 네이버 금융 파싱 + V5.2 퀀트 로직 |
| `dashboard.html` | 프론트엔드 대시보드 (서버가 자동으로 서빙) |

두 파일은 **반드시 같은 폴더**에 두세요.

---

## 실행 방법

### 1. 라이브러리 설치
```
pip install flask flask-cors requests beautifulsoup4 lxml
```

### 2. 서버 실행
```
python proxy_server.py
```
정상 실행되면 터미널에 `http://localhost:8000` 안내가 출력됩니다.

### 3. 접속
브라우저에서 **http://localhost:8000** 으로 접속하면 대시보드가 열립니다.

### 4. 분석 실행
- **네이버 금융 모드** (기본): API 키 없이 바로 사용 가능. 거래량 상위 종목을 자동 스캔합니다.
- **키움 REST API 모드**: APP Key / Secret Key 입력 시 키움 데이터로 분석.
- 파라미터(K값, 노이즈 임계값, 목표 수익률, 손절 기준) 조정 후 **V5.2 자동 분석 실행** 클릭.

서버가 꺼져 있으면 **데모 모드**로 샘플 데이터 로직 시연이 가능합니다.

## V5.2 퀀트 통합 로직 — 5단계 공정

1. **팩트 데이터 수집** — 네이버 금융에서 실제 시세(시가·고가·저가·종가)를 오차 없이 파싱. 가짜 숫자 개입 차단.
2. **주도 테마 필터링 + 압축** — 거래대금 상위 종목 추림 → 노이즈 지수 & Cpk로 가장 깨끗하고 강한 2종목 압축.
   - `Noise = 1 − |Open − Close| / (High − Low)`
3. **매수 타점 산출** — 래리 윌리엄스 변동성 돌파, 고정 K값 0.2.
   - `Target = Open + (Prev_High − Prev_Low) × 0.2`
4. **조건부 진입** — 실시간 주가가 Target 돌파 시에만 매수, 미달 시 관망.
5. **무결점 리스크 관리** — 익절 / 손절 / Zero Overnight(장 마감 전 전량 청산) 중 하나로 당일 종료.

## 백테스트 기능

대시보드 하단의 **백테스트** 섹션에서 과거 데이터에 V5.2 로직을 재현할 수 있습니다. 두 가지 모드가 있습니다.

### 일봉 모드 (키 불필요, 최대 4년)
- 네이버 금융 일봉으로 4년까지 백테스트
- 빠르지만 장중 익절·손절 도달 순서를 알 수 없어, 둘 다 닿은 날은 **익절 우선** 가정 (낙관적 편향)

### 분봉 모드 (키움 키 필요, 최근 1~3개월)
- 키움 REST API `ka10080`(주식분봉차트조회)으로 분봉 수집
- 분봉을 시간순으로 따라가며 익절가·손절가 중 **실제로 먼저 닿은 것**을 청산으로 확정 → '익절 우선' 가정이 사라져 훨씬 정확
- 키움 APP Key / Secret Key 필요. 종목·날짜마다 API를 호출하므로 느림 (2개월·12종목 기준 5~15분)
- 키움 조회 제한 때문에 기간을 짧게 (1~3개월) 권장

공통으로 자산 곡선, 연간/월간 수익률, 승률, MDD, 샤프 지수, 청산 사유 분포, 체결 내역을 제공하며, 거래세·수수료·슬리피지를 매 거래 차감합니다.

**백테스트의 한계 (반드시 인지)**
- **일봉 모드 — 익절 우선 가정**: 하루 안에 익절가·손절가에 둘 다 닿은 날은 익절로 처리합니다. 낙관적이라 결과가 실제보다 좋게 나오는 편향이 있습니다. (분봉 모드는 이 한계가 없음)
- **분봉 모드 — 1분봉 내부 순서**: 한 분봉 안에서 고가·저가 순서는 알 수 없어, 한 봉에서 둘 다 닿으면 보수적으로 손절 처리합니다. 그래도 일봉보다 비교할 수 없이 정밀합니다.
- **생존 편향 (두 모드 공통)**: 종목 유니버스가 '현재 시점' 거래량 상위로 고정됩니다. 과거 각 날짜의 실제 거래량 순위는 복원 불가합니다. **분봉 모드로도 해결되지 않습니다.**
- **4단계 호가창 매수세 미반영**: 봉 데이터로는 Target 돌파 여부만 판정합니다.
- 위 한계들 때문에 백테스트 수익률이 실제 운용 수익률을 보장하지 않습니다.

## 실전 기록장

대시보드 상단의 **📅 실전 기록장** 탭에서 매일 추천받은 종목과 그 결과를 누적해서 추적할 수 있습니다.

### 사용 흐름
1. **시드머니 설정** — 기록장 탭 상단에서 실제 운용 시드 입력 (기본 100만원)
2. **오늘의 추천 분석** — 추천 탭에서 V5.2 분석 실행 → 추천 카드 확인
3. **추천 저장** — 추천 카드 아래 **📔 오늘 추천 기록장에 저장** 버튼 클릭
4. **장 마감 후 결과 입력** — 기록장 탭에서 각 종목의 **결과 입력** 클릭 → 익절/손절/시간청산/관망 선택 → 청산가 입력
5. **누적 확인** — 시드 대비 누적 자산·수익 금액, 월별 수익률, 자산 곡선이 자동 계산됨

### 계산 방식
- 매일 시드를 **2종목에 50:50 균등 배분**해 거래한다는 가정
- 하루 손익률 = (종목1 순수익률 + 종목2 순수익률) / 2
- 거래세(0.18%) · 수수료(0.015%) · 슬리피지(0.1%)를 차감한 **순수익률** 기준
- 월별 누적은 **복리** 계산 — (1+r1)×(1+r2)×…×(1+rN) − 1

### 추천 카드 형식
저장된 추천은 다음 8개 항목으로 구성됩니다.
1. 종목명·종목코드
2. 상승확률 (노이즈·Cpk 종합 점수를 백분율로 환산한 것이며, 진짜 통계적 확률이 아닙니다)
3. 현재가 (전일 종가)
4. 구매가 (계산식 표시: `Open + (Prev_High − Prev_Low) × 0.2`)
5. 목표가
6. 손절가
7. 기대 수익률
8. 퀀트 분석 선정 사유 — **지표 부분만 자동 생성**되며, **이슈 부분은 직접 입력**해야 합니다 (자동 분석은 가짜 정보를 만들 위험이 있어 의도적으로 비워둠)

### 데이터 저장 위치
서버 폴더의 `journal.json` 파일에 저장됩니다. Render에 배포한 경우 무료 플랜은 디스크가 휘발성이라 일정 시간 후 초기화될 수 있습니다. 영구 보존이 필요하면 Render Disk 옵션(유료)을 사용하거나, PC에서 직접 실행하는 방식이 안전합니다.

## 키움 REST API 키 발급 (키움 모드 사용 시)

1. https://openapi.kiwoom.com 접속 → 우측 상단 **API 사용신청**
2. 약관 동의 후 신청 → **APP Key / Secret Key** 다운로드
3. **계좌 APP Key 관리**에서 본인 PC 공인 IP 등록 (필수)
4. 대시보드의 키움 모드에 APP Key, Secret Key 입력

## 주의사항

- 네이버 금융 파싱 데이터는 **지연 시세**일 수 있습니다. 실거래 전 반드시 실시간 시세 확인이 필요합니다.
- 4단계 "조건부 진입"의 호가창 매수세 판단은 자동 분석에 포함되지 않으므로 별도 확인이 필요합니다.
- 본 시스템은 참고용 분석 도구이며, 모든 투자 결정과 결과는 본인 책임입니다.
- 네이버 금융을 과도하게 빠르게 요청하면 일시 차단될 수 있어, 서버에 요청 간격(0.15초)을 두었습니다. 스캔 종목 수가 많으면 수십 초 소요됩니다.
- 클라우드 배포 시(예: Streamlit Cloud, Render 등)에는 `proxy_server.py`의 `app.run()` 호스트/포트와 `dashboard.html`의 `PROXY` 주소를 환경에 맞게 수정하세요.
